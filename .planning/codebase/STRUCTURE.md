## Directory Purposes

**`client/`:**
- Purpose: Everything that runs in the Electron renderer process (the visible application)
- Contains: HTML shell, CSS, Preact components, P2P logic, local storage/DB modules
- Key files: `client/src/app.tsx` (root), `client/app.html` (HTML entry), `client/style.css` (all styles)

**`client/src/components/`:**
- Purpose: Preact functional components that make up the UI
- Contains: 5 component files — one per major UI section
- Key files: `client/src/components/Chat.tsx`, `client/src/components/Voice.tsx`, `client/src/components/Sidebar.tsx`

**`client/src/lib/`:**
- Purpose: Core logic modules — singletons for networking, input, and storage utilities
- Contains: Service classes (signaling, WebRTC, PTT), storage helpers (localStorage, IndexedDB), types, platform detection
- Key files: `client/src/lib/webrtc.ts` (P2P connections), `client/src/lib/signaling.ts` (relay WS), `client/src/lib/store.ts` (settings/rooms)

**`client/dist/`:**
- Purpose: Build output directory — bundled JS and copied CSS files
- Generated: Yes (by `scripts/build.ts` and `scripts/dev.ts`)
- Committed: No (gitignored)

**`electron/`:**
- Purpose: Electron main process code and application icons
- Contains: Main process entry, preload bridge, type definitions, compiled JS output, icon assets
- Key files: `electron/main.ts` (app entry), `electron/preload.ts` (IPC bridge)

**`electron/icons/`:**
- Purpose: Application and system tray icons for all platforms
- Contains: SVG sources, ICO (Windows), ICNS (macOS), multi-resolution PNGs
- Committed: Yes

**`comms-relay/`:**
- Purpose: Self-contained signaling relay server — deployable independently
- Contains: Single-file Bun WebSocket server, tests, Docker config, static landing page
- Key files: `comms-relay/index.ts` (server), `comms-relay/index.test.ts` (integration tests)

**`scripts/`:**
- Purpose: Build and development tooling scripts
- Contains: Dev server with watch mode, production build script, Bun build plugin
- Key files: `scripts/dev.ts` (dev workflow), `scripts/build.ts` (production build)

**`data/`:**
- Purpose: Local SQLite database files (for development/testing)
- Generated: Yes
- Committed: No (gitignored via `*.db` pattern)

**`docs/`:**
- Purpose: Project documentation, screenshots, demo video, landing page
- Committed: Yes

## Key File Locations

**Entry Points:**
- `electron/main.ts`: Electron main process — app lifecycle, window creation, IPC
- `client/src/app.tsx`: Preact renderer — root `<App />` component, `render()` call
- `client/app.html`: HTML shell — loaded by BrowserWindow, links CSS + scripts
- `comms-relay/index.ts`: Signaling server — Bun HTTP+WebSocket server
- `scripts/dev.ts`: Development workflow — build, watch, restart Electron
- `scripts/build.ts`: Production build — bundle + minify + copy assets

**Configuration:**
- `package.json`: Root package — scripts, dependencies, `"main": "electron/main.js"`
- `tsconfig.json`: TypeScript — Preact JSX, ESNext target, bundler resolution
- `electron-builder.yml`: Packaging — asar, platform targets, file includes, icons
- `comms-relay/package.json`: Relay package — separate from root, zero runtime deps

**Core Logic:**
- `client/src/lib/webrtc.ts`: WebRTC peer management (479 lines — largest file)
- `client/src/lib/signaling.ts`: WebSocket signaling with auto-reconnect
- `client/src/lib/ptt.ts`: Push-to-talk handler (DOM + Electron hooks)
- `client/src/lib/store.ts`: Settings, rooms, invite encoding/decoding
- `client/src/lib/chatdb.ts`: IndexedDB chat persistence
- `client/src/lib/sounds.ts`: Web Audio API notification sounds

**Testing:**
- `client/src/lib/store.test.ts`: Unit tests for invite encode/decode + signaling URL
- `client/src/lib/signaling.integration.test.ts`: Integration test with ephemeral WS server
- `comms-relay/index.test.ts`: Relay integration tests (spawns server subprocess)

**Styles:**
- `client/style.css`: All application CSS (656 lines, 5 CSS layers: base, utils, components, responsive, app)

**Type Definitions:**
- `electron/electron.d.ts`: `ElectronAPI` interface + `Window` augmentation (main-side)
- `client/src/lib/electron.d.ts`: Same interface for renderer-side TypeScript
- `client/src/lib/store.ts`: `User`, `Room`, `ChatMessage`, `Settings`, `VoiceMode` types

## Naming Conventions

**Files:**
- Components: `PascalCase.tsx` (e.g., `Chat.tsx`, `Voice.tsx`, `ScreenShare.tsx`)
- Library modules: `lowercase.ts` (e.g., `signaling.ts`, `webrtc.ts`, `chatdb.ts`)
- Tests: `{module}.test.ts` or `{module}.integration.test.ts` — co-located with source
- Type definitions: `{context}.d.ts` (e.g., `electron.d.ts`)
- Build scripts: `lowercase.ts` (e.g., `dev.ts`, `build.ts`)

**Directories:**
- All lowercase, no separators: `components`, `lib`, `scripts`, `icons`
- Feature-grouped: `client/`, `electron/`, `comms-relay/` at root level

**Exports:**
- Components: named exports (`export function Chat(...)`)
- Singletons: named export of instance (`export const signaling = new SignalingClient()`)
- Utility functions: named exports (`export function encodeInvite(...)`)
- Types: named exports (`export interface User { ... }`)
- No default exports used anywhere in the codebase
- No barrel files (`index.ts` re-exports) — all imports reference specific files

## Where to Add New Code

**New UI Component:**
- Create: `client/src/components/MyComponent.tsx`
- Export: `export function MyComponent({ ... }: MyComponentProps)`
- Import in: `client/src/app.tsx` or parent component
- Style in: `client/style.css` under the `@layer components` section
- Follow pattern: functional component with Preact hooks, WebTUI attributes via JSX spread syntax

**New Core Logic Module:**
- Create: `client/src/lib/mymodule.ts`
- Pattern: If stateful, create a class and export a singleton (`export const myModule = new MyModule()`)
- Pattern: If stateless, export named functions
- Tests: `client/src/lib/mymodule.test.ts` (co-located)

**New IPC Channel:**
1. Add handler in `electron/main.ts` inside `registerIpc()`: `ipcMain.handle('my-channel', ...)`
2. Expose in `electron/preload.ts` via `contextBridge.exposeInMainWorld()`
3. Add type to `electron/electron.d.ts` and `client/src/lib/electron.d.ts`
4. Optionally wrap in `client/src/lib/ipc.ts`

**New Signaling Message Type:**
1. Add `case 'my_type':` in relay's `message()` handler in `comms-relay/index.ts`
2. Handle in client's `signaling.on('my_type', ...)` listener
3. Add test case in `comms-relay/index.test.ts`

**New Settings Field:**
1. Add to `Settings` interface in `client/src/lib/store.ts`
2. Add localStorage read in `store.settings` initialization
3. Add localStorage write in `saveSettings()`
4. Add UI control in `client/src/components/Settings.tsx`

**New Test File:**
- Client unit test: `client/src/lib/{module}.test.ts`
- Client integration test: `client/src/lib/{module}.integration.test.ts`
- Relay test: Add to `comms-relay/index.test.ts` describe block
- Run: `bun test` (uses `bun:test` runner)
